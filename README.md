# ST Card Builder

面向 SillyTavern 的浏览器端角色卡、世界书与辅助工作流编辑器。项目基于 Astro，编辑结果保存在浏览器草稿中，可导入、导出并继续在 SillyTavern 中使用。

## 功能

- 角色卡基础信息编辑与预览。
- 内嵌世界书条目的创建、编辑、搜索、AI 扩写与触发词整理。
- AI 配置、预设导入，以及按现有上下文生成角色和世界书骨架。
- 世界书审计与本地试聊。
- 变量卡、正则和状态栏相关编辑能力。
- 草稿切换、浏览器本地保存、JSON 导入与导出。
- 小说转换器 V2：导入长文本后按片段抽取事实，支持可配置并行数、断点保存和失败重试；最终生成世界书草稿，预览确认后才会写入正式世界书。

## 小说转换器 V2

小说转换器将 AI 的语义理解与程序的结构控制分开：

1. 选择“剧情延续”或“世界观重建”模式。
2. 选择要发布为独立世界书条目的类别：人物、地点、势力、物品、规则、事件、关系。
3. 导入 `.txt`、`.md` 或 `.json` 文本，设置单片大小、并行任务数与严格质量检查。
4. 系统按章节优先切片，分片并行抽取结构化事实并保存任务状态。
5. 合并事实后生成世界书草稿，并在本地清洗触发词、过滤泛词和未选择类别。
6. 在预览中选择应用全部有效条目、仅实体类、仅规则类或勾选的条目。

实体型条目默认按绿灯层级生成，基础规则类条目按蓝灯层级生成。第一版不默认生成 MVU 变量、剧情门控或自动解锁规则。应用前会保存一次世界书快照，支持撤销本次注入。

## 开发

要求：Node.js 18 或更高版本。

```bash
git clone git@github.com:wominIII/st-card-builder.git
cd st-card-builder
npm install
npm run dev
```

默认开发地址为 `http://localhost:4321`。

## 构建

```bash
npm run build
npm run preview
```

`npm run build` 会生成静态文件到 `dist/`，可直接部署到任意静态 Web 服务。项目当前部署示例使用 Caddy 提供静态文件，并保留既有的 `/api/*` 反向代理配置。

## AI 接口

在页面的 AI 配置区域填写兼容 OpenAI Chat Completions 的接口地址、密钥和模型。密钥只由浏览器请求时使用；请不要将密钥写入仓库或导出的角色卡。

## 数据说明

- 编辑状态主要保存在浏览器 `localStorage` 和当前角色卡扩展数据中。
- 小说转换草稿不会在用户点击应用前写入正式世界书。
- 应用小说草稿只会替换本工坊生成的 `[小说...]` 条目，普通手动世界书条目会保留。

## 验证

```bash
npm run build
```

当前项目尚未配置自动化测试套件；提交前至少执行一次构建验证。
